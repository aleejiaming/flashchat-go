🧗 本次遭遇的困難與成長覆盤

這次的更新重點在於「擴增基礎設施（Redis）」，雖然程式碼行數不多，但在閱讀與理解上的挑戰非常大。

痛點一：Go 語言的命名方式與函數閱讀

我的盲點：常常被 Go 的函數名稱或命名慣例搞糊塗，花了很多時間在「閱讀文件」上。

突破與體悟：隨著對 Go 的熟悉度上升，我發現它其實比 Java 友善。Go 的邏輯非常直白，沒有太多隱藏的黑魔法。只要順著資料流（心智圖），就能看懂八成的運作邏輯。

痛點二：第三方套件的使用障礙 (Input / Output)

我的盲點：在使用官方或第三方套件（例如 WebSocket 的 WriteJSON、Redis 的 LRange / LTrim 等）時，往往需要花非常多時間去理解「這個方法到底要吃什麼資料型態？又會回傳什麼結果給我是 error 還是陣列？」。

突破觀念：這是強型別語言的必經之路。未來的解法是：不要怕看原始碼。遇到不會的函數，直接在 IDE 裡點進去看它的「簽名 (Signature)」，例如 func WriteJSON(v interface{}) error，一眼就能看出它吃任意型態，並回傳錯誤。
